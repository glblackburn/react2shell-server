## Documentation Maintenance

### Adding New Documentation

1. **Choose the right category:**
   - Active documentation ‚Üí appropriate category directory
   - Historical/archived ‚Üí `archive/` or `historical/`
   - Analysis reports ‚Üí `analysis/`
   - Test-related ‚Üí `testing/`

2. **Follow naming conventions:**
   - Use descriptive names with underscores or hyphens
   - Include dates for time-sensitive documents (YYYY-MM-DD)
   - Use consistent capitalization

3. **Update this README:**
   - Add entry to appropriate category section
   - Update quick links if it's a key document
   - Add to documentation index

4. **Add metadata:**
   - Include "Last Updated" date
   - Mark status (Active, Historical, Planning, Analysis)
   - Add cross-references to related documents
